Documentation

Research

All three websites that I chose to analyse had minimal breakpoints but were still creative and complex with their layout and content. I believe this is the key to a well-made site. It doesn't need to be overly complicated with many breakpoints to be responsive and functional. While researching websites I noticed that they a lot of them had similar layouts with hero content as the main element then a cascading layout consisting of the main content.

design choices

With the research in mind I chose to have a minimal layout consisting of a horizontal navbar, a hero image with text and box layout for my research (which could be news or other types of content). The responsive elements of the website are the navbar changing to a dropdown menu for small screen sizes and the main content changing from a three column layout to a single column layout.

usability considerations

The website is minimal with a clear structure which remains responsive between different websites.

challenges encountered

The challenges I faced included whether I should have used the flex or grid syntax to layout my website. The grid and flex syntax can help create a fluid responsive design system by organising content and allowing the site to become more streamlined. The grid syntax is better for vertical layout and flex is better for horizontal layout. I chose to go with a grid layout for the header and main content of the website because a grid layout is better for vertical content layout. Additionally I faced challenges with where to layout my content and the variable font not centring on the baseline properly (help Ben why is this happening?! If you highlight the text it doesn't align properly and it's annoying).
